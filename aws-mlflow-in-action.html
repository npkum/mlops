Predicting Heart Disease using Machine Learning


Goal here is to build a machine learning model to predict heart disease and cpature the life-cycle of ML model using Mlflow

Machine Learning steps:

  • Exploratory data analysis (EDA) - the process of going through a dataset and finding out more about it.
  • Model training - create model(s) to learn to predict a target variable based on other variables.
  • Model evaluation - evaluating a models predictions using problem-specific evaluation metrics.
  • Model comparison - comparing several different models to find the best one.
  • Model fine-tuning - once we've found a good model, how can we improve it?
  • Feature importance - since we're predicting the presence of heart disease, are there some things which are more important for prediction?
  • Cross-validation - if we do build a good model, can we be sure it will work on unseen data?
  • Productionizing machine learning models - deploy model in production using MLflow

1. Problem Definition


Given clinical parameters about a patient, can we predict whether or not they have heart disease?

The problem in this ML project is we will be exploring the binary classification. Using differnet features about a patient to predict whether they have heart disease or not.

2. Data


Interrogate the data for the problem definition. This involves sourcing, defining different parameters and discussing with SMEs and finding out expected outcome



The data for this project is sourced from the Cleveland database from UCI Machine Learning Repository.

Attributes and features are also referred to as independent variables and a target variable can be referred to as a dependent variable.



We use the independent variables to predict our dependent variable. In this experiment, the independent variables are a patients different medical attributes and the dependent variable is whether or not they have heart disease

3. Evaluation


Set the success criteria by defining the evaluation metric at the start of a project.



Since machine learning is very experimental, with the help of SME you might set go nogo as below:
If we can reach 85% accuracy at predicting whether or not a patient has heart disease during the proof of concept, we'll pursure this project.

This provides a rough goal for a machine learning engineer or data scientist to work towards.



However, its important to note that the evaluation metric may change over time due to changes in data and introduction new medicines and treatments.

#set model threshold score
r2Threshold=float(0.80)

# Create models in a dictionary
models = {"KNN": KNeighborsClassifier(),
          "Logistic Regression": LogisticRegression(max_iter=1000), 
          "Random Forest": RandomForestClassifier()}

# Set multiple tags at once
tags = {"reMark": "Compare ML Algorithms",
        "runVersion": "1.0.0"}

# Set model stage status
stage = "Staging" # values for <stage>: Staging|Archived|Production|None

# Random seed for reproducible results
np.random.seed(42)

# create a list to store model scores
model_scores = {}

for name, model in models.items():
    #print("Model name:",model)
    with mlflow.start_run(run_name=name) as run:
        model_name = name
        mlflow.set_tags (tags) #set multiple tags at once
        mlflow.sklearn.autolog()
        run_id = run.info.run_uuid
        # Fit the model to the data
        model.fit(X_train,y_train)
        # Make preidctions on test data
        y_preds = model.predict(X_test)
        # Log Models With Signatures
        #signature = infer_signature(X_train, logreg.predict(X_train))
        signature = infer_signature(X_train, model.predict(X_train))
              
        # Evaluate the model and append its score to model_scores
        model_scores[name] = model.score(X_test, y_test)
        acc = model.score(X_test, y_test)
        mlflow.log_metric("testing_data_accuracy", acc) #additional logging of scores
        
        # Register the model/s only if it meets the threshold
        reg_model_name = experiment_name + '-' + model_name
        if acc >= r2Threshold:
            mlflow.sklearn.log_model(sk_model=model, artifact_path = "model", registered_model_name=reg_model_name
                                     , signature=signature)
            print(" Model met threshold and registred was :", model)
            # Transition the current model to 'Staging'
            client = MlflowClient()
            # Get all versions of the model filtered by name
            model_name = reg_model_name
            filter_string = "name='{}'".format(model_name)
            results = client.search_model_versions(filter_string)
            for res in results:
                current_version = res.version
                desc = "This model is a version of scikit-learn" + model_name + ".This model met the threshold"                
            client.transition_model_version_stage(name=model_name, stage=stage, version=current_version) #stage & version
            client.update_model_version(name=model_name, version=current_version, description=desc) # add/update description
    mlflow.end_run()

Hyperparameter tuning


Hyperparameter tunning will performed on only those algorithms with best model scores. In this experiment it will performed on LogisticRegression and RandomForestClassifier
  • Tuning models with with RandomizedSearchCV : tries n_iter combinations of hyperparameters and saves the best.
  • Tuning a model with GridSearchCV : tries every single combination of hyperparameters and saves the best.

To test different hyperparameters, one could use a validation set. Since the data used in this experiment ~300 records, we'll use cross-validation.
The most common type of cross-validation is k-fold. It involves splitting data into k-fold's and then testing a model on each

#set model threshold score
hypThreshold=float(0.88)

# Create hyperparameter dictionary
hyper_Dict = {1: {'ml-algorithm':'LogisticRegression', 'tunning-method':'RandomizedSearchCV'},
              2: {'ml-algorithm':'LogisticRegression', 'tunning-method':'GridSearchCV'},
              3: {'ml-algorithm':'RandomForest', 'tunning-method':'RandomizedSearchCV'}}

# create a hyperparameter grid (a dictionary of different hyperparameters) for each and then test them out.
log_reg_grid = {"C": np.logspace(-4, 4, 20), "solver": ["liblinear"]}

# Different RandomForestClassifier hyperparameters
rf_grid = {"n_estimators": np.arange(10, 1000, 50), "max_depth": [None, 3, 5, 10],
           "min_samples_split": np.arange(2, 20, 2), "min_samples_leaf": np.arange(1, 20, 2)}

# Create dataframe to store best hyperparamaters & score
LR_hyp = pd.DataFrame(columns=['ml-algorithm','tunning-method', 'best-parms', 'score'])

# Set random seed
np.random.seed(42)

# Set multiple tags at once
tags = {"reMark": "hyperparameter-tunning-cv",
        "runVersion": "1.0.0"}

# Set model stage status
stage = "Staging" # values for <stage>: Staging|Archived|Production|None

print("Started hyperparameter tunning on selected ml algorithms, this may take some time.....")
for name, hyper_Dict in hyper_Dict.items():
    #print("run-name:", hyper_Dict['ml-algorithm'])  
      
    if hyper_Dict['ml-algorithm'] == 'LogisticRegression' and hyper_Dict['tunning-method'] == 'RandomizedSearchCV':
        #mlflow_run_name = hyper_Dict['ml-algorithm']+ '-' +hyper_Dict['tunning-method']
        mlflow_run_name = 'Hyperparameter-LR'+ '-' +hyper_Dict['tunning-method']
        with mlflow.start_run(run_name = mlflow_run_name) as run:
            mlflow.set_tags (tags) #set multiple tags at once
            # enable autologging
            mlflow.sklearn.autolog()
            # Setup random hyperparameter search for LogisticRegression
            rs_log_reg = RandomizedSearchCV(LogisticRegression(), param_distributions=log_reg_grid, cv=5, n_iter=20, verbose=True)
            # Fit random hyperparameter search model
            rs_log_reg.fit(X_train, y_train)
            # Log Models With Signatures
            signature = infer_signature(X_train, rs_log_reg.predict(X_train))
            best_parms = rs_log_reg.best_params_
            score = rs_log_reg.score(X_test, y_test)
            LR_hyp = LR_hyp.append({'ml-algorithm':hyper_Dict['ml-algorithm'], 'tunning-method':hyper_Dict['tunning-method'], 'best-parms':best_parms, 'score':score},ignore_index=True)
            
            # Register the model if it meets the threshold
            if float(score) >= hypThreshold:
                print(" Model met threshold and registred was :",  mlflow_run_name)
                reg_model_name = experiment_name + '-' + mlflow_run_name
                mlflow.sklearn.log_model(sk_model=name, artifact_path="model",registered_model_name=reg_model_name
                                         ,signature=signature)
                # Transition the current model to 'Staging'
                client = MlflowClient()
                # Get all versions of the model filtered by name
                model_name = reg_model_name
                filter_string = "name='{}'".format(model_name)
                results = client.search_model_versions(filter_string)
            for res in results:
                current_version = res.version
                desc = "This model is a version of scikit-learn" + model_name + ".This model met the threshold"
            client.transition_model_version_stage(name=model_name,stage=stage,version=current_version) #add/update stage & version
            client.update_model_version(name=model_name, version=current_version, description=desc) #add/update description
        mlflow.end_run()
         
    elif hyper_Dict['ml-algorithm'] == 'LogisticRegression' and hyper_Dict['tunning-method'] == 'GridSearchCV':
        #mlflow_run_name = hyper_Dict['ml-algorithm'] + '-' + hyper_Dict['tunning-method']
        mlflow_run_name = 'Hyperparameter-LR'+ '-' +hyper_Dict['tunning-method']
        with mlflow.start_run(run_name = mlflow_run_name) as run:
            mlflow.set_tags (tags) #set multiple tags at once
            # enable autologging
            mlflow.sklearn.autolog()
            # Setup random hyperparameter search for LogisticRegression
            gs_log_reg = GridSearchCV(LogisticRegression(), param_grid=log_reg_grid, cv=5, verbose=True)
            # Fit random hyperparameter search model
            gs_log_reg.fit(X_train, y_train)
            # Log Models With Signatures
            signature = infer_signature(X_train, gs_log_reg.predict(X_train))                             
            best_parms = gs_log_reg.best_params_
            score = gs_log_reg.score(X_test, y_test)
            LR_hyp = LR_hyp.append({'ml-algorithm':hyper_Dict['ml-algorithm'], 'tunning-method':hyper_Dict['tunning-method'], 'best-parms':best_parms, 'score':score},ignore_index=True)

            cv_recall = np.mean(cross_val_score(clf, X, y, cv=5, scoring="recall"))
            # Cross-validated F1 score
            cv_f1 = np.mean(cross_val_score(clf,X, y, cv=5, scoring="f1"))    
            cv_metrics = pd.DataFrame({"Accuracy": cv_acc, "Precision": cv_precision,
                                        "Recall": cv_recall, "F1": cv_f1}, index=[0])
            # Log Models With Signatures
            signature = infer_signature(cross_val_predict(clf, X, y, cv=5))
            mdl_metrics = mdl_metrics.append({'ml-algorithm':hyper_eval['ml-algorithm'], 'Accuracy':cv_acc, 'Precision':cv_precision, 'Recall':cv_recall, 'F1':cv_f1},ignore_index=True)
            # Fit an instance of LogisticRegression (taken from above)
            clf.fit(X_train, y_train)
            # Register the model if it meets the threshold
            if (float(cv_acc) >= accThreshold) and (float(cv_precision) >= precThreshold) and (float(cv_f1) >= f1Threshold) and (float(cv_recall) >= recallThreshold):
                print(" Model met threshold and registred was :", mlflow_run_name)
                reg_model_name = experiment_name + '-' + mlflow_run_name
                # Register the model
                mlflow.sklearn.log_model(sk_model=clf, artifact_path="model", registered_model_name=reg_model_name
                                         , signature=signature)
                #mlflow.sklearn.log_model(sk_model=name, artifact_path="model", registered_model_name=reg_model_name)
                # Transition the current model to 'Production'
                client = MlflowClient()
                # Get all versions of the model filtered by name
                model_name = reg_model_name
                filter_string = "name='{}'".format(model_name)
                results = client.search_model_versions(filter_string)
                for res in results:
                    current_version = res.version
                    desc = "This model is a version of scikit-learn" + model_name + ".This model met the threshold"
                    client.transition_model_version_stage(name=model_name,stage=stage,version=current_version) #stage & version
                    client.update_model_version(name=model_name, version=current_version, description=desc) #add/update description
        mlflow.end_run()

  • Precision - Indicates the proportion of positive identifications (model predicted class 1) which were actually correct. A model which produces no false positives has a precision of 1.0.
  • Recall - Indicates the proportion of actual positives which were correctly classified. A model which produces no false negatives has a recall of 1.0.
  • F1 score - A combination of precision and recall. A perfect model achieves an F1 score of 1.0.
  • Accuracy - The accuracy of the model in decimal form. Perfect accuracy is equal to 1.0.
